Summary

What changed vs. today

Structural changes

  • Dashboard becomes the home page. Replaces today's empty "Welcome." Shows active engagements, signals, recent activity.
  • Property overview becomes the default property route. Brand DNA moves to its own tab. /properties/[slug] now lands on an at-a-glance summary.
  • Property hero header replaces the small left-aligned banner. Phase strip with labels, 4 key metrics, breadcrumb.
  • Tab cluster separators visually group always-on tabs vs. project-driven tabs vs. meta tabs.
  • Inheritance banner on Brand DNA — explicit "inheriting N sections from X" with Override Locally per section.
  • Action histogram on Pages tab doubles as a filter. Undecided rows highlighted.
  • Projects render as cards with phase strips, not table rows.

Visual changes

  • Refined neutral palette — zinc/slate hierarchy instead of Tailwind defaults. Single accent (near-black, with indigo for active states).
  • Tabular numerics on stat values, page counts, ranks.
  • Sidebar adds search (⌘K), groups Properties under collapsible clients with phase pills.
  • Status pills standardized — Active emerald · Prospect sky · Paused amber · Inactive slate · Completed indigo. Each with a colored dot prefix.
  • Action pills on pages get their own palette to avoid confusion with status pills.
  • Pipeline phase strips appear consistently across dashboard, client detail, property hero, project cards.
  • Activity feed as a recurring pattern (signal dot + title + meta line + timestamp).

What this doesn't change: the data model. Same client → property → project hierarchy. Same Supabase tables. Same BQ Meta integration. The Vercel Python functions for clients / projects / competitors still drive the data — the React server components just render it more deliberately. This is purely a frontend reorganization; nothing in the Python or schema layer would need to move.
